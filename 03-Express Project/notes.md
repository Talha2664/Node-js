Body → POST (mostly)
```
Use it When Data is large or sensitive
Rule:
When you SEND data → use BODY
Example use:  
Login form
Signup form
```

Query → GET
```
Data is small
Needs to be visible in URL 
Used to filter or search
Rule:
When you ASK data → use QUERY
Example use:
Search
Filters
Pagination
```

Dynamic Route → GET (mostly)
```
To identify one specific item
Clean & readable URL
Example use:
Get user by ID
Get product by ID
Rule:
When you TARGET something → use DYNAMIC ROUTE
```

GET:
```
Used to ask / fetch data
Data visible in URL
Small amount of data only
Examples: search, filter, read user info
```

POST:
```
Used to send / create data
Data hidden in body
Can be large and secure
Examples: signup, login, add product
```
